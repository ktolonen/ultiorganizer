This help page focuses on managing an event. The menu items you see depend on your rights, but the task flow below assumes full administrator access.

There are two main management areas in the left menu:

  • The Administration menu contains installation-level tools that affect the whole system.
  • Each event also has its own administration menu with tools for managing that specific event.

A typical setup order is:

  1. Create the event.
  2. Add divisions.
  3. Add teams.
  4. Create pool rule templates and pools.
  5. Create field locations and reservations.
  6. Generate games.
  7. Schedule games.
  8. Review standings, pool moves, and other event data during the event.

Administration menu

  • Events - Create and manage events.
  • Rule templates - Maintain reusable game and pool rule templates.
  • Clubs & Countries - Manage shared club and country data used by teams and players.
  • Field locations - Create venues and playing sites.
  • Field reservations - Create the playable time slots used for scheduling.
  • Translations - Manage translated user-facing texts when translation rights are available.
  • Users - Manage user accounts, rights, and registration requests.
  • API Tokens - Manage API access tokens.
  • Logs - Review system and event activity logs.
  • Database - Access database maintenance and tools.
  • Settings - Manage installation-wide settings.

Event administration menu

  • Event - Event overview and core event settings.
  • Divisions - Create and maintain the divisions in the event.
  • Teams - Add and manage event teams.
  • Pools - Create and maintain pools, placement pools, and pool moves.
  • Scheduling - Schedule games into reservations.
  • Games - Review and edit generated or scheduled games.
  • Spirit - Manage spirit settings and spirit submissions in spirit-enabled events.
  • Standings - Review and correct standings before confirming moves when needed.
  • Season points - Manage event scoring tables when season points are enabled.
  • Accreditation - Review player accreditation and related checks.
  • Game responsibilities - Review the games assigned to responsible users.
  • Contacts - Access organizer and team contact information.

Event setup

  • Use Events to create a new event or edit an existing one.
  • Set the event name, dates, type, organizer, and any enrollment or spirit-related options needed for that event.
  • After the event has been created, its management links appear in the left menu.
  • The event page is the main entry point for divisions, teams, pools, scheduling, games, standings, accreditation, contacts, and spirit tools when enabled.

Divisions and pools

  • Use Divisions to create the competition structure for the event.
  • Each division should be created before teams are enrolled or approved for it.
  • Use Pools to create the playing groups or stages inside a division.
  • Pool order, visibility, continuation settings, placement settings, and rules all affect later standings, moves, and scheduling.
  • When using continuation or placement pools, check the order carefully, because it affects how final standings are calculated.
  • If teams move from one pool to another, use the pool move tools only after the source results are correct.

Teams and players

  • Use Teams to add teams directly or to review enrolled teams.
  • If the event uses enrollment, create divisions first and then approve or manage teams from the event tools.
  • Team pages are also where team responsibilities, player lists, and related team data are managed.
  • Player and accreditation tasks belong under the team and accreditation views once teams have been assigned correctly.

Users and roles

  • Use Users to add user accounts, review registration requests, and assign rights.
  • Add an event administrator when someone needs full event management rights for one event.
  • Add a spirit administrator when someone needs to manage spirit settings, submissions, and spirit review tools.
  • Add a team administrator when someone needs to manage one team's players and team data.
  • Add game input responsibilities when someone needs to enter results or other game data for selected games or reservations.
  • After assigning rights, check that the correct event administration links appear in the left menu for that user.

Locations and scheduling

  • Use Field locations to create the venues or playing sites.
  • Use Field reservations to define the actual playable time slots on those locations.
  • Use Scheduling to place games into reservations after games have been generated.
  • If a game needs to be changed later, edit it from the games view or return to scheduling when you need to move several games.
  • Make sure teams, pools, game lengths, and reservations are ready before scheduling, because changes later can create extra cleanup work.

Games and standings

  • Generate games only after the division, pool, and team setup is ready.
  • For continuation pools, scheduling names may be shown until the teams are moved in.
  • Use Games to review, edit, reschedule, or delete games when needed.
  • Use Standings to review pool rankings and to correct standings before confirming pool moves.
  • If standings or moves look wrong, fix the earlier result or pool data first and then confirm the moves again.

Other management tools

  • Accreditation is used for player accreditation and related checks.
  • Contacts gives access to event organizers and team contact details.
  • Spirit is available in spirit-enabled events for spirit settings, review, and missing submission follow-up.
  • Rule templates, Clubs & Countries, Users, API Tokens, Logs, Database, and Settings are installation-level administration tools.
